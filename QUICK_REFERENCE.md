## ğŸ› ï¸ Quick Workflows

### Policy Development
1. Define problem using consilience
2. Research across disciplines (with AI)
3. Engage diverse stakeholders
4. Design solution with anti-corruption measures
5. Build in transparency and accountability
6. Use [Policy Proposal Template](templates/policy-proposal.md)

### Facilitate Discussion
1. Set ground rules for respect
2. Build shared understanding
3. Map multiple perspectives
4. Generate solutions collaboratively
5. Evaluate fairly
6. Use [Discussion Framework](templates/discussion-framework.md)

### Use AI Effectively
1. Humans set goals and values
2. AI assists with research/analysis
3. Verify important claims
4. Apply human judgment
5. Document transparently
6. See [AI Deliberation Guide](resources/ai-deliberation-guide.md)

### Build Transparency
1. Default to open
2. Proactive disclosure
3. Accessible formats
4. Timely publication
5. Complete and accurate
6. See [Transparency Guidelines](resources/transparency-guidelines.md)

### Prevent Corruption
1. Eliminate monopoly power
2. Limit discretion
3. Maximize accountability
4. Ensure transparency
5. Eliminate conflicts of interest
6. See [Anti-Corruption Measures](resources/anti-corruption-measures.md)

## ğŸ’¡ Key Concepts

**Consilience**: Integration of knowledge from different disciplines to create comprehensive understanding.

**Human-Guided AI**: Humans set values and make decisions; AI assists with research and analysis.

**Radical Transparency**: Default to open; secrecy requires justification.

**Anti-Corruption by Design**: Build prevention into systems, not just punishment after.

**Inclusive Participation**: All stakeholders have meaningful opportunity to participate.
